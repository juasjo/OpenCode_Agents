# Reglas del proyecto

- Cuando el usuario pida crear código, primero aclara lo mínimo imprescindible solo si falta un dato crítico.
- Si la petición ya es suficiente, no pidas más contexto y propone una solución razonable.
- Para tareas de programación:
  1. analizar objetivo
  2. proponer plan breve
  3. implementar
  4. revisar
  5. proponer tests
- Explica de forma breve y práctica.
- Prioriza scripts pequeños y útiles.
- Para Python:
  - usar Python 3
  - añadir comentarios cortos
  - evitar dependencias externas salvo que aporten mucho
  - indicar cómo ejecutar el script
